MNIST Digit Recognition System Implementation Plan
This document outlines the proposed implementation plan to satisfy the requirements for your Machine Learning Assignment 2 (Handwritten digit recognition).

Goal Description
We will develop a PyTorch-based Deep Learning pipeline to read the MNIST dataset, prepare custom train/val/test splits, construct multiple models (Feedforward and Convolutional), and conduct an extensive analysis on hyperparameter changes (learning rate, batch size, neurons, and network depth).

The result will be a suite of Python scripts that automatically run these experiments, generate all required plots (loss/accuracy curves), build a confusion matrix, and output analytical insights.

Proposed Changes
We will create a modular Python project to keep the code clean, easily testable, and maintainable.

Core Implementation Framework
[NEW] data_loader.py
Setup custom dataset loading logic depending on our dataset choice (Kaggle CSV or Torchvision).
Apply scikit-learn train_test_split twice in a stratified manner to achieve 60% Training, 20% Validation, 20% Test.
Create torch.utils.data.DataLoader instances for the splits.
[NEW] models.py
FNN Model: A customizable Feedforward Neural Network using PyTorch, accepting layer architectures dynamically to facilitate the layer/neurons analysis required in Step 4. Uses strictly ReLU activations.
CNN Model (Bonus): A Convolutional Neural Network incorporating Convolutional layers, Max Pooling, Dropout, and Layer Normalization.
[NEW] trainer.py
Implement building blocks for the custom training loop.
Track metrics using SGD optimizer and Cross-Entropy loss.
Return training and validation accuracy/loss history for plotting.
Analysis & Experiments Logic
[NEW] utils.py
Helper functions to plot Training & Validation metrics using matplotlib (saving them to plots/ directory).
Helper function to generate and plot the Confusion Matrix using seaborn and sklearn.metrics.
[NEW] main.py
The core script orchestrating all experiments:
Base Model Run (LR=0.01, Default batch, 2 Hidden layers)
LR Analysis (4 distinct learning rates: e.g., 0.1, 0.05, 0.01, 0.001)
Batch Size Analysis (4 batch sizes: e.g., 16, 32, 64, 128)
Neurons Analysis (4 variations in neuron counts)
Layer Count Analysis (e.g., 2, 3, 4, 5 hidden layers)
Final Test Set Evaluation & Confusion Matrix.
Bonus Base Model Run (CNN + LayerNorm + Dropout).
A corresponding report.md will be generated with textual insights and metrics for each step.
